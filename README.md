# Cash Flow Manager

## Описание

Приложение позволяет вести учет операций прихода и расхода денежных средств с привязкой к статусу, типу, категории и подкатегории. Пользователь может создавать, редактировать, удалять и просматривать записи по ДДС, а также управлять справочниками статусов, типов, категорий и подкатегорий.

## Реализованные возможности

- Просмотр списка записей ДДС
  - Таблица со столбцами: дата, статус, тип, категория, подкатегория, сумма, комментарий
  - Фильтрация по: диапазону дат, статусу, типу, категории и подкатегории
- Создание записи о движении денежных средств
  - Дата создания записи заполняется автоматически, но может быть изменена вручную
  - Статус (например: бизнес, личное, налог и т.д.)
  - Тип (например: пополнение, списание и т.д.)
  - Категория (например: инфраструктура, маркетинг и т.д.)
  - Подкатегория (например: VPS, Proxy, Farpost и т.д.)
  - Сумма
  - Комментарий (необязательно)
- Редактирование записи ДДС
- Удаление записи ДДС
- Управление справочниками
  - Добавление, редактирование и удаление статусов
  - Добавление, редактирование и удаление типов
  - Добавление, редактирование и удаление категорий
  - Добавление, редактирование и удаление подкатегорий
- Логические принадлежности
  - Подкатегории привязаны к категориям
  - Категории связаны с типами
- Валидация данных
  - Проверка на стороне клиента:
    - Поля `status`, `type`, `category`, `subcategory`, `amount` обязательны к заполнению
    - Поле `amount` проверяется на положительное значение (>=0)
  - Проверка на стороне сервера:
    - Проверяется соответствие категории к выбранному типу и подкатегории к выбранной категории
    - Проверяется положительность введенной суммы

## Архитектура приложения

- `manage.py` - главный по запуску Django проекта
- `config/` - настройки проекта, ключ, настройки для БД и т.д.
- `cashflow/` - приложение с моделями, формами, представлениями и шаблонами
- `cashflow/migrations/` - миграции для БД
- `cashflow/static/cashflow/css/styles.cc` - простенькие css стили для более приятного восприятия интерфейса
- `cashflow/templates/cashflow/` - html страницы:
  - `list.html` - главная страница с таблицей
  - `directory.html` - страница со справочником по статусам, типам, категориям и подкатегориям
  - `form.html` - форма для создания/изменения различных данных
- `cashflow/admin.py` - стандартная админка от django
- `cashflow/forms.py` - модели форм для валидации и заполнения данных
- `cashflow/models.py` - модели БД:
  - `Status`
  - `Type`
  - `Category`
  - `SubCategory`
  - `CashFlow`
- `cashflow/urls.py` - эндпоинты
- `docker-compose.yml` - конфигурация контейнеров для приложения и базы данных
- `Dockerfile` - сборка Docker-образа

## Запуск проекта

### 1. Установка зависимостей, если запуск производится без Docker

Проект содержит зависимости в `requirements.txt`.

Если вы запускаете без Docker, то установите зависимости для python (используемая версия python в проекте: 3.12):

```bash
pip install -r requirements.txt
```

### 2. Настройка окружения

В корне проекта создайте файл `.env` и присвойте необходимые параметры. Пример параметров:

```env
SECRET_KEY="super_puper_secret_key"
DEBUG=True

NAME_DB="cashflow_db"
USER_DB="user"
PASSWORD_DB="password"
PORT_DB=5432
```

### 3. Запуск через Docker

В корне репозитория выполните команду:

```bash
docker-compose up --build -d
```

### 4. Ручный запуск без Docker

Проект содержит зависимости в `requirements.txt`.

Если вы запускаете без Docker, то установите зависимости для python (используемая версия python в проекте: 3.12):

```bash
pip install -r requirements.txt
```
А так же настройте и запустите базу данных PostgreSQL. Если база данных PostgreSQL запущена и работает, то выполните команду:

```bash
python manage.py migrate
python manage.py runserver
```

## Использование

- Откройте в браузере `http://127.0.0.1:8000/` для просмотра списка записей ДДС
- Используйте кнопку `Создать запись` для добавления новой операции
- Используйте фильтры по дате, статусу, типу, категории и подкатегории
- Используйте кнопку `Зайти в справочник` для перехода на страницу для управления статусами, типами, категориями и подкатегориями

## Важные нюансы

- Категория привязана к типу, поэтому нельзя выбрать категорию, не относящуюся к выбранному типу
- Подкатегория привязана к категории, поэтому нельзя выбрать подкатегорию, не относящуюся к выбранной категории
- Нельзя удалить статус, тип, категорию или подкатегорию, если существует зависимая запись ДДС или если существуют зависимые категория→тип и подкатегория→категория

## Примечания

- В текущей реализации используется PostgreSQL, а не SQLite
- Админ-панель Django доступна по адресу `http://127.0.0.1:8000/admin/`
